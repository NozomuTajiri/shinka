# 第14章 構造を変える

人を責めるな、構造を変えろ

---

## 1. 成功の後の違和感

ナカムラ工業とカワシマ精密工業での成功から二週間が経った五月の第二週、山田は営業部の定例会議で部下たちの報告を聞いていた。

佐藤課長が、今週の案件状況を説明している。

「以上、五件の提案中、三件が受注、二件が商談継続中です」

「良い数字だな」山田は頷いた。「受注率六割は立派だ」

若手社員たちも、明るい表情を浮かべている。ナカムラ工業での五千万円の受注以来、営業部全体の雰囲気が変わっていた。

「部長の新しい提案手法が、効いてますね」佐藤が言った。「リスク回避の価値を前面に出す方法、本当に響きます」

「ああ」山田は答えた。

しかし、その声には、わずかな違和感が混じっていた。

会議が終わり、部下たちが散っていく。山田は一人、ホワイトボードの前に立ったまま、何かを考え込んでいた。

「部長、何か気になることでも？」

佐藤が声をかけてきた。

「佐藤、一つ聞きたいんだが」

「はい」

「今週、受注できなかった二件。誰の担当だ？」

「えっと......」佐藤は資料を確認した。「高橋と、田中です」

「そうか」

山田は、二人の名前を聞いて、ある思いが心に浮かんだ。

高橋も田中も、営業部では中堅クラスだ。決してサボっているわけではない。むしろ、真面目に働いている。それなのに、受注率が低い。

「佐藤、彼らは何が足りないと思う？」

佐藤は少し考えてから答えた。

「やる気......ではないと思います。二人とも頑張ってますから」

「じゃあ、何だ？」

「スキル、でしょうか。部長のように、お客様の課題を見抜いて、価値を提案する力が、まだ不足しているのかもしれません」

山田は、その答えに何か引っかかるものを感じた。

スキル。

確かに、それはあるかもしれない。でも、それは本当に「彼らの問題」なのだろうか。

---

## 2. 宮本からの助言

その日の午後、山田は宮本のもとを訪れた。

珈琲を淹れながら、宮本が聞いた。

「最近、調子が良いそうじゃないか。ナカムラ工業に続いて、カワシマ精密工業も」

「ありがとうございます」山田は言った。「ただ、少し気になることがありまして」

「ほう」

山田は、営業部の状況を説明した。受注できる社員と、できない社員の差。同じ研修を受け、同じ提案手法を教えているのに、結果に大きな開きがある。

「それで、君はどう思っているんだ？」宮本が聞いた。

「正直......彼らのスキルが足りないのかな、と」

「ふむ」宮本はコーヒーカップを置いた。「山田君、一つ質問していいか」

「はい」

「君の部下が成果を出せないとき、それは誰の責任だと思う？」

山田は答えに窮した。

「それは......本人の、責任......でしょうか」

宮本は首を横に振った。

「違う」

「じゃあ、誰の......」

「構造の問題だ」

「構造......？」

宮本はノートを開き、大きく書いた。

```
【組織の鉄則】
人を責めるな、構造を変えろ
```

「山田君、君は今、『高橋と田中のスキルが足りない』と考えている。つまり、個人の問題だと捉えている」

「はい......」

「でも、それは間違いだ」宮本は真剣な目で言った。「個人を責めても、何も変わらない。変えるべきは、『構造』なんだ」

---

## 3. なぜ同僚は抵抗するのか

「構造、とは具体的に何ですか？」

山田が聞くと、宮本はホワイトボードに図を描き始めた。

```
【営業部の構造】

評価制度
    ↓
行動
    ↓
結果
```

「組織には、必ず『構造』がある。評価制度、プロセス、文化——こうしたものが、人の行動を決める」

「評価制度が、行動を決める......」

「そうだ」宮本は説明を続けた。「例えば、君の営業部では、何で評価される？」

山田は即答した。

「売上です。四半期ごとの売上目標を達成したかどうか」

「それだけか？」

「基本的には、はい」

宮本は頷いた。

「では、質問だ。もし君が『リスク回避の価値提案』を教えて、部下がそれを実践したとする。ただし、その提案は時間がかかる。従来の値引き営業なら一週間で決まる案件が、三週間かかるかもしれない」

「そうですね......顧客の課題を深く理解する必要がありますから」

「その場合、部下はどう考えるだろう？」

山田は、はっとした。

部下の立場で考えてみる。

四半期の売上目標が課されている。目標未達なら、評価が下がる。ボーナスも減る。

従来のやり方なら、確実に数字が作れる。リスクが低い。

一方、新しいやり方は、時間がかかる。しかも、うまくいくかどうか分からない。

「部下は......新しいやり方を避けるかもしれません」

「その通り」宮本が力強く言った。「これが、『構造の問題』だ」

---

## 4. 評価制度の問題

宮本は、ホワイトボードに新しい図を書いた。

```
【構造が生む行動】

評価制度: 四半期売上のみ
    ↓
社員の行動: 短期的な数字を追う
    ↓
結果: 値引き営業、顧客関係の軽視
```

「見てくれ。評価制度が『四半期売上のみ』だと、社員は短期的な数字を追うようになる」

「はい......」

「値引きをすれば、すぐに受注できる。時間をかけて顧客の課題を理解するよりも、手っ取り早い」

山田は、自分自身がかつてそうだったことを思い出した。

四半期の終わりが近づくと、焦って値引きをしていた。目標達成のために、利益を削ってでも数字を作っていた。

「つまり、高橋や田中が新しいやり方をしないのは、彼らが悪いのではなく......」

「評価制度が悪いんだ」宮本が断言した。

山田は、深く息を吐いた。

「じゃあ、どうすればいいんですか？」

「構造を変える」宮本はシンプルに答えた。「評価制度、プロセス、文化——社員が『正しい行動を取りたくなる』構造に変えるんだ」

---

## 5. 小さな構造変革の提案

「具体的には、どう変えればいいんでしょうか」

山田が聞くと、宮本はノートに書き始めた。

```
【評価制度の見直し案】

Before: 四半期売上のみで評価
    ↓
After: 複数の指標で評価

1. 売上（40%）
2. 顧客満足度（30%）
3. 提案の質（20%）
4. 長期的な顧客関係（10%）
```

「まず、評価軸を増やす。売上だけで評価するのではなく、顧客満足度、提案の質、長期的な関係構築も評価に含める」

山田は、メモを取りながら聞いた。

「顧客満足度は、どう測るんですか？」

「簡単だ。四半期ごとに、顧客にアンケートを送る。『営業担当の対応に満足していますか』『提案は役に立ちましたか』——こうした質問をして、点数化する」

「なるほど......」

「提案の質も同じだ。上司が部下の提案書をレビューし、『お客様の課題を理解しているか』『価値を明確に伝えているか』を評価する」

宮本は続けた。

「こうすれば、社員は『ただ売上を上げる』だけでは評価されない。『お客様に本当に価値を提供しているか』が問われる」

「それは......確かに、行動が変わりそうですね」

「そうだ」宮本は頷いた。「人は、評価される項目を最適化する。だから、評価制度を変えれば、行動が変わる」

---

## 6. プロセスの問題

宮本は、次のページを開いた。

「評価制度だけじゃない。プロセスも変える必要がある」

「プロセス......？」

「そうだ。例えば、君の営業部では、提案書を作るとき、上司のレビューはあるか？」

山田は考えた。

「基本的には、ありません。各自が自分で作って、そのまま顧客に出しています」

「それが問題だ」

宮本はホワイトボードに書いた。

```
【提案プロセスの見直し】

Before:
個人で提案書を作成 → 顧客に提出

After:
個人で提案書を作成 → 上司レビュー → 顧客に提出
```

「提案書は、必ず上司がレビューする。『お客様の課題を理解しているか』『価値が明確か』『伝わる言葉で書かれているか』——これらをチェックする」

「でも、それだと時間がかかりませんか？」

「かかる」宮本は認めた。「でも、質の低い提案を出して失注するより、時間をかけて質の高い提案を出す方が、長期的には効率がいい」

山田は、納得した。

確かに、今までは「数を打てば当たる」という考えで、とにかく多くの提案を出していた。でも、受注率は低かった。

「さらに」宮本は続けた。「レビューの過程で、部下は学ぶ。『どう書けば伝わるのか』『どう考えればお客様の課題が見えるのか』——これが、スキルの底上げになる」

---

## 7. 田村専務への進言

翌日、山田は田村専務のもとを訪れた。

「専務、ご相談があります」

「何だ？」

山田は、宮本から学んだことを説明した。

評価制度の問題。プロセスの問題。そして、『人を責めるのではなく、構造を変える』という考え方。

田村は、真剣に聞いていた。

「つまり、君は営業部の評価制度を変えたいと？」

「はい」山田は資料を広げた。「現在の『四半期売上のみ』の評価から、複数指標での評価に変えたいんです」

田村は資料を見た。

```
【新しい評価制度】

1. 売上（40%）
2. 顧客満足度（30%）
3. 提案の質（20%）
4. 長期的な顧客関係（10%）
```

「顧客満足度、か」田村が呟いた。「これは、どう測る？」

「四半期ごとに、アンケートを実施します。既に質問項目も作成しました」

山田は、別の資料を見せた。

```
【顧客満足度アンケート】

1. 営業担当の対応に満足していますか？（5段階評価）
2. 提案内容は、御社の課題解決に役立ちましたか？（5段階評価）
3. 今後もこの営業担当と取引したいですか？（5段階評価）
```

田村は、じっくりと資料を読んだ。

「提案の質、というのは？」

「上司が部下の提案書をレビューし、評価します。基準も明確にしました」

山田は、評価基準を示した。

```
【提案の質 評価基準】

1. 顧客の課題を正確に理解しているか（25点）
2. 価値を明確に伝えているか（25点）
3. 顧客の言葉で書かれているか（25点）
4. 具体的な数字で効果を示しているか（25点）

合計100点満点
```

田村は、腕を組んで考え込んだ。

「山田君」

「はい」

「この新しい評価制度、いつから始めたい？」

山田は驚いた。てっきり、反対されると思っていた。

「え......できれば、次の四半期から」

「分かった」田村が言った。「やってみろ」

「本当ですか!?」

「ただし、条件がある」

「何でしょうか」

「半年で結果を出すこと。受注率、顧客満足度、そして売上——すべてで改善を示してくれ」

「はい。必ずやります」

---

## 8. 営業部への説明

月曜日の朝、山田は営業部全員を集めた。

「今日は、重要な発表がある」

部下たちが、緊張した面持ちで山田を見た。

「次の四半期から、評価制度を変える」

ざわめきが起こった。

「これまでは、四半期売上だけで評価していた。でも、これからは違う」

山田は、ホワイトボードに書いた。

```
【新しい評価制度】

1. 売上（40%）
2. 顧客満足度（30%）
3. 提案の質（20%）
4. 長期的な顧客関係（10%）
```

「見てくれ。売上はもちろん重要だ。でも、それだけじゃない。顧客満足度、提案の質、長期的な関係——これらも評価する」

若手社員が手を挙げた。

「部長、これだと評価が厳しくなりませんか？」

「逆だ」山田が答えた。「今までは、売上が上がらなければ、どんなに頑張っても評価されなかった。でも、新しい制度では、売上が低くても、顧客満足度が高ければ評価される」

佐藤課長が補足した。

「つまり、『正しい営業』をすれば、必ず評価されるということですね」

「その通り」山田が頷いた。「値引きして無理やり受注するんじゃなく、お客様の課題を理解して、価値を提供する。それが評価される」

高橋が、不安そうに聞いた。

「部長、僕みたいに受注率が低い人間は、どうすればいいんですか？」

山田は、高橋の目を見た。

「高橋、君は悪くない」

「え......？」

「受注できないのは、君のせいじゃない。俺たちが、正しいやり方を教えてこなかったからだ」

会議室がシンと静まった。

「これからは、ちゃんと教える。提案書は、必ず俺か佐藤がレビューする。『どう書けば伝わるのか』『どう考えればお客様の課題が見えるのか』——一緒に学んでいこう」

高橋の目に、涙が浮かんだ。

「ありがとうございます......」

---

## 9. 新しいプロセスの導入

その日の午後、佐藤が最初の提案書レビューを依頼してきた。

「部長、田中の提案書です。見ていただけますか？」

山田は、提案書を受け取った。

タイトルは「コスト削減提案」。

一ページ目から、違和感があった。

```
【Before】

当社の高性能検査装置を導入することで、
貴社の検査コストを削減できます。

装置仕様:
- 精度: 0.01mm
- 速度: 50個/分
- 価格: 1,200万円
```

山田は、赤ペンを取り出した。

「田中、ちょっと来てくれるか」

田中が緊張した面持ちで近づいてきた。

「この提案書、読んでみてどう思う？」

「え......と、技術仕様をしっかり書いたつもりなんですが」

「うん、仕様は書いてある。でも、お客様の課題は？」

「課題......？」

「そうだ。この提案を受け取ったお客様は、『自分の課題が解決できる』と思うか？」

田中は、提案書を読み直した。

「......思わないかもしれません」

「なぜだと思う？」

「お客様の課題が、書かれていないからです」

「その通り」山田は頷いた。「じゃあ、一緒に書き直そう」

---

## 10. 一緒に学ぶ

山田と田中は、会議室で向かい合った。

「まず、このお客様の課題は何だ？」

「えっと......検査に時間がかかって、生産が遅れているそうです」

「良い。それが課題だ。じゃあ、それを最初に書こう」

二人は、提案書を書き直し始めた。

```
【After】

御社の検査工程では、現在1個あたり3分かかっており、
これが生産ボトルネックになっています。

その結果、
- 納期遅延が月2回発生
- 残業時間が月80時間増加
- 機会損失: 月間約300万円

私たちの提案する検査装置を導入すると、
検査時間が3分から30秒に短縮されます。

つまり、
- 納期遅延ゼロ
- 残業時間60%削減
- 月間300万円の機会損失を回収

初期投資1,200万円は、
わずか4ヶ月で回収できます。
```

田中は、Before と After を見比べた。

「全然......違いますね」

「そうだろう？」山田が言った。「Before は『製品の説明』。After は『お客様の課題解決』だ」

「はい......」

「田中、君は悪くない。ただ、やり方を知らなかっただけだ」

山田は、田中の肩に手を置いた。

「これからは、俺たちが教える。一緒に学んでいこう」

「ありがとうございます」

田中の顔に、初めて笑顔が浮かんだ。

---

## 11. 三ヶ月後の変化

新しい評価制度とプロセスを導入して、三ヶ月が経った。

営業部の数字が、明らかに変わっていた。

```
【Before（3ヶ月前）】
受注率: 25%
顧客満足度: 測定なし
平均提案作成時間: 2日
四半期売上: 2.8億円

【After（現在）】
受注率: 48%
顧客満足度: 平均4.2/5.0
平均提案作成時間: 4日（レビュー含む）
四半期売上: 3.6億円
```

受注率が、ほぼ倍になっていた。

田村専務が、山田を呼んだ。

「山田君、素晴らしい結果だ」

「ありがとうございます」

「提案作成に時間がかかるようになったのに、売上が増えている。これは、どういうことだ？」

山田は答えた。

「質が上がったからです。従来は、10件提案して2件受注していました。でも今は、7件提案して4件受注できます」

「なるほど......」

「時間をかけて、お客様の課題を理解する。そして、価値を明確に伝える。それができれば、受注率は必ず上がります」

田村は、満足そうに頷いた。

「評価制度を変えた効果は？」

「大きいです」山田は資料を見せた。「以前は、部下たちは『とにかく数を出せ』と考えていました。でも今は、『質の高い提案を出そう』と考えるようになりました」

「顧客満足度の効果は？」

「これが一番大きいかもしれません」

山田は、別の資料を広げた。

「顧客満足度を測り始めてから、部下たちが『お客様の声』を気にするようになりました。『この提案、お客様は満足してくれるだろうか』と考えるようになった」

「構造が、行動を変えたわけか」

「はい。宮本先生の言う通り、『人を責めるのではなく、構造を変える』——これが、すべてでした」

---

## 12. 高橋の成功

ある日、高橋が山田のデスクに駆け寄ってきた。

「部長！　やりました！」

「どうした？」

「サトウ電機から、受注いただきました！　2,500万円です！」

山田は立ち上がった。

「本当か!?　おめでとう！」

サトウ電機は、高橋が三ヶ月間追いかけていた案件だった。何度も提案書を書き直し、山田と一緒にレビューを重ねた。

「部長、本当にありがとうございます」高橋が深々と頭を下げた。「三ヶ月前の僕なら、絶対に取れなかった案件です」

「君が頑張ったからだ」

「いえ」高橋は首を振った。「部長が、ちゃんと教えてくれたからです。提案書の書き方、お客様の課題の見つけ方、価値の伝え方——全部、レビューで学びました」

山田は、胸が熱くなった。

高橋は、決して能力が低いわけではなかった。ただ、正しいやり方を知らなかっただけだ。

そして、今までの評価制度では、彼は報われなかった。短期的な売上を求められ、焦って値引きをして、それでも受注できず、評価が下がる。

## 本章のキーメッセージ

**人を責めるな、構造を変えろ**

組織の問題は、ほとんどの場合、個人の問題ではなく構造の問題である。

人は、評価される通りに行動し、報酬が得られる通りに動く。

従来の評価制度:
- 四半期売上のみで評価
- 結果: 短期的な数字追求、値引き営業、顧客関係の軽視

新しい評価制度:
- 売上（40%）+ 顧客満足度（30%）+ 提案の質（20%）+ 長期的な顧客関係（10%）
- 結果: 正しい営業の実践、受注率向上、組織文化の改善

構造を変えれば、行動が変わる。
行動が変われば、結果が変わる。
結果が変われば、文化が変わる。

リーダーの仕事は、部下を叱ることではない。
部下が成功できる『仕組み』を作ることだ。
