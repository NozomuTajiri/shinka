# 第18章「マーケットインの思想」

世界初・業界初はニーズの深掘りから生まれる

---

## 1. 新商品開発会議

「では、次期主力商品の開発方針について、技術部門からプレゼンテーションをお願いします」

社長の言葉を受けて、技術部長の佐藤が立ち上がった。会議室のスクリーンに、最新技術を駆使した製品のコンセプト図が映し出される。

「我が社の最先端AI技術を活用した、業界初のスマート家電シリーズを提案します。IoT技術とAIを組み合わせ、家電同士が連携して学習する革新的なシステムです」

佐藤の説明は熱を帯びていた。開発チームが半年かけて練り上げた技術の粋を集めた製品だ。しかし、山田は違和感を覚えていた。

「素晴らしい技術だとは思いますが」山田は慎重に口を開いた。「お客様はそれを求めているのでしょうか？」

会議室に一瞬、沈黙が流れる。佐藤の表情が硬くなった。

「山田さん、これは業界初の技術です。お客様はまだこの便利さを知らないだけです。一度使えば、その価値がわかるはずです」

「それは本当でしょうか」山田は穏やかだが、はっきりと言った。「『業界初』は手段であって、目的ではありません。お客様が抱えている課題は何か。その解決策として、この技術が最適なのか。そこから考える必要があるのではないでしょうか」

## 2. プロダクトアウトの罠

営業部長の田中が山田に同調した。

「実は、先月お客様にヒアリングを実施したんです。高機能な家電よりも、『シンプルで使いやすいもの』『故障が少ないもの』『価格が手頃なもの』という声が圧倒的に多かったんです」

佐藤は反論した。

「しかし、それでは競合と差別化できません。技術力で勝負しなければ、価格競争に巻き込まれるだけです」

「佐藤さんの言いたいことはわかります」山田は頷いた。「でも、それは『プロダクトアウト』の発想なんです。私たちが持っている技術をどう製品化するか、という考え方です」

山田は立ち上がり、ホワイトボードに図を描き始めた。

```
【プロダクトアウト】
技術・製品 → 顧客
「この技術が素晴らしいから買ってください」

【マーケットイン】
顧客ニーズ → 解決策 → 技術・製品
「お客様の課題を解決するために、最適な技術を選択する」
```

「私たちが目指すべきは『マーケットイン』の発想です」山田は参加者を見渡した。「お客様のニーズを深く理解し、そこから逆算して製品を設計する。技術はあくまで手段です」

## 3. 真のニーズの発見

「では、山田さん」社長が身を乗り出した。「お客様の真のニーズとは何だと考えますか？」

山田は一呼吸置いた。これは重要な問いだ。

「実は、先週から顧客訪問を重ねてきました。表面的には『シンプルな家電がほしい』と言われます。しかし、その裏にある本当のニーズは違うんです」

山田はスマートフォンを取り出し、録音した顧客インタビューを再生した。

『うちの母親、80歳なんですけどね。リモコンのボタンが多すぎて使えないんですよ。せっかく高い家電を買ったのに、基本機能しか使ってない。もったいないですよね』

別の声が続く。

『共働きで忙しいんです。家電の設定とか、マニュアル読むのが面倒で。買ったその日から、誰でも簡単に使える製品がほしいんです』

さらに別の声。

『一人暮らしの息子に家電を送りたいんだけど、使い方を教えられない。遠距離だから。遠くからでもサポートできる製品ってないですかね』

会議室が静まり返った。

「聞こえてきたのは」山田は言った。「『技術的な高機能』ではなく、『人に寄り添う製品』へのニーズです。高齢者、忙しい共働き世帯、遠距離の家族。それぞれが抱える『使いにくさ』という課題です」

## 4. ニーズからの逆算

佐藤の表情が変わった。技術者としてのプライドと、新しい視点への興味が交錯している。

「つまり、高機能である必要はない、と？」

「いいえ」山田は首を振った。「必要な機能は高度に。でも、使い方は究極にシンプルに。それが真のイノベーションだと思うんです」

山田はホワイトボードに書き始めた。

```
【真のニーズ】
• 高齢者でも迷わず使える
• 設定不要で即使える
• 遠隔サポートが可能
• 故障の予兆を検知
• 必要な機能だけを提供

【技術的解決策】
• AIによる使用者学習
• 音声ガイダンス
• リモートサポート機能
• 予防保守システム
• パーソナライズUI
```

「佐藤さんが提案されたAI技術」山田は佐藤を見た。「それを、この方向で活用できませんか？家電が使用者を学習し、その人に最適な使い方を提案する。複雑な機能は隠して、必要な機能だけを表示する」

佐藤の目が輝いた。技術者として、新しいチャレンジが見えてきたのだ。

「それなら、音声インターフェースも組み合わせられる。『おはよう』と言えば、その人の朝のルーティンに合わせて家電が動く」

「そうです！」山田は興奮を抑えきれなかった。「技術的には高度だけど、使う側は『おはよう』と言うだけ。これこそが、真のイノベーションではないでしょうか」

## 5. 画期的アイデアの誕生

社長が立ち上がった。

「面白い。では、具体的な商品コンセプトに落とし込んでみましょう。ブレインストーミングをしましょう」

会議室に活気が戻ってきた。

営業の田中が発言する。

「『家族見守り機能』はどうでしょう。遠く離れた高齢の親が家電を使った履歴を、子供のスマホに通知する。いつも通りに生活していれば安心。異常があればすぐわかる」

企画の鈴木が続ける。

「『成長する家電』というコンセプトは？使う人に合わせて進化していく。最初は基本機能だけ。慣れてきたら、『こんな使い方もできますよ』と提案する」

佐藤が技術的な裏付けを加える。

「エッジAIを使えば、個人データをクラウドに送らずに学習できる。プライバシーも守れます」

山田は議論を整理していく。

「つまり、私たちが開発するのは『人に寄り添う家電シリーズ』。キャッチコピーは『あなただけの、成長する家電』。どうでしょう？」

社長が大きく頷いた。

「素晴らしい。これなら、技術力も活かせるし、お客様のニーズにも応えられる。価格競争にも巻き込まれない」

しかし、山田はさらに一歩進めた。

「もう一つ提案があります。この製品を『売り切り』にするのではなく、『サービス』として提供するのはどうでしょうか」

## 6. ビジネスモデルの革新

全員が山田に注目した。

「どういうことですか？」社長が問う。

「家電を売るのではなく、『快適な暮らし』を売るんです。月額制のサブスクリプションモデル。故障したら無料交換、アップデートも無料、24時間サポート付き」

CFOの石井が懸念を示した。

「それでは利益が出ないのでは？」

「いえ、むしろ逆です」山田は数字を示した。「従来モデルでは、一度売ったら終わり。でも、サブスクなら継続収入が見込めます。そして、お客様との関係が続く。次の商品開発にも、リアルタイムの使用データが活かせます」

山田はさらに続けた。

「初期費用が抑えられるので、若い世代も購入しやすい。高齢者向けには『家族プラン』を用意して、離れて暮らす子供が親のために契約できる。企業向けには『福利厚生プラン』として提案できます」

佐藤が驚きの声を上げた。

「それは、家電業界の常識を覆すモデルですね」

「そうです」山田は力強く言った。「でも、お客様のニーズから逆算すると、このモデルが最適解なんです。『壊れたら困る』『高額な買い替えは避けたい』『いつでもサポートがほしい』。そんな声に応えるには、このビジネスモデルしかありません」

## 7. マーケットインの本質

社長が深く頷いた。

「山田君、君が言う『マーケットイン』の本質が見えてきました。それは単に『お客様の声を聞く』ということではないんですね」

「はい」山田は答えた。「表面的な要望だけでなく、その裏にある本当の課題を見つける。そして、その課題を解決するために、技術もビジネスモデルも、すべてを最適化する。それがマーケットインの思想です」

山田はこれまでの経験を振り返った。コンサルタント時代、多くの企業が「技術はあるのに売れない」という悩みを抱えていた。それは、技術起点で製品を作っていたからだ。

「プロダクトアウトが間違っているわけではありません」山田は参加者を見渡した。「技術革新も重要です。でも、その技術を『誰のため』『何のため』に使うのか。そこを見失うと、独りよがりな製品になってしまいます」

佐藤が真剣な表情で言った。

「私も反省しました。技術者として、最新技術を使うことに夢中になっていました。でも、本当に大切なのは『その技術で誰を幸せにできるか』ですね」

「そうです」山田は微笑んだ。「技術は手段です。目的は、お客様の課題解決、お客様の幸せです。そこをブレさせてはいけません」

## 8. イノベーションの定義

企画の鈴木が質問した。

「でも、山田さん。マーケットインだと、お客様が想像できる範囲の製品しか生まれないのでは？本当のイノベーションは、お客様が知らない価値を提供することではないですか？」

それは本質的な問いだった。スティーブ・ジョブズの有名な言葉がある。「顧客は、それを見せてあげるまで、自分が何を欲しいのかわからない」

山田は頷いた。

「良い質問です。確かに、お客様は解決策を知らないかもしれない。でも、課題は持っているんです。私たちがすべきは、お客様も気づいていない深層のニーズを見つけ出すこと」

山田は例を挙げた。

「iPodが登場する前、誰も『1000曲をポケットに』とは言いませんでした。でも、『音楽をもっと手軽に持ち運びたい』というニーズはあった。ジョブズは、そのニーズの本質を見抜いたんです」

「つまり」鈴木が理解を深める。「マーケットインとは、お客様の『言葉』をそのまま製品にするのではなく、その裏にある『本質的なニーズ』を見抜くことなんですね」

「その通りです」山田は強調した。「そして、その本質的ニーズに対して、お客様が想像もしなかった解決策を提示する。それが真のイノベーションです」

## 9. 深掘りの技術

営業の田中が興味深そうに聞いた。

「山田さんは、どうやって本質的ニーズを見つけているんですか？何かコツがあるんですか？」

山田は考えながら答えた。

「『なぜ？』を5回繰り返すんです。トヨタの『5 Why』という手法ですね」

山田はホワイトボードに例を書いた。

```
【表面的な声】
「シンプルな家電がほしい」

【深掘り】
なぜ？→「機能が多すぎて使いこなせない」
なぜ？→「マニュアルを読むのが面倒」
なぜ？→「時間がない、理解するのが難しい」
なぜ？→「自分に必要な機能だけ知りたい」
なぜ？→「自分の使い方に合わせてほしい」

【本質的ニーズ】
→「私に最適化された、成長する製品」
```

「このように」山田は説明した。「表面的な『シンプルさ』の裏には、『パーソナライゼーション』のニーズがあったんです。これがわかれば、AI技術の活用という解決策が見えてきます」

佐藤が感心したように言った。

「技術者として、お客様と直接話す機会が少なかった。これからは、もっとお客様の声を聞きに行きます」

「ぜひそうしてください」山田は励ました。「技術者だからこそ、お客様の課題を聞いたときに、『この技術で解決できる』というアイデアが浮かぶはずです」

## 10. 世界初の意味

社長が総括した。

「今日の議論で明確になったことがあります。『世界初』や『業界初』は、目的ではなく結果だということです」

社長は参加者を見渡した。

「お客様の本質的ニーズを深く理解し、それを解決するために最適な技術とビジネスモデルを組み合わせる。その結果として、誰もやっていない『世界初』が生まれる」

山田が続けた。

「『世界初』は手段ではありません。お客様のニーズに真摯に向き合った結果、自然と生まれるものです。そして、そういう『世界初』こそが、市場で受け入れられ、成功するんです」

CFOの石井が実務的な確認をした。

「では、この『人に寄り添う家電シリーズ』のプロジェクト化を進めてよろしいですか？」

社長が力強く答えた。

「承認します。山田君、プロジェクトリーダーをお願いできますか？」

山田は一瞬驚いたが、すぐに決意を固めた。

「はい、お引き受けします」

## 11. プロジェクトの始動

会議後、佐藤が山田に声をかけてきた。

「山田さん、最初は反発してしまって申し訳ありませんでした。でも、今日の議論で目が覚めました」

「いえ」山田は笑顔で応えた。「佐藤さんの技術への情熱があったからこそ、良いアイデアが生まれたんです。これから、一緒に最高の製品を作りましょう」

佐藤が真剣な表情で言った。

「今まで、技術を製品にすることばかり考えていました。でも、本当は『誰を幸せにするか』から始めるべきだったんですね」

「技術者としてのプライドは大切です」山田は言った。「でも、その技術が人の役に立ってこそ、本当の価値が生まれる。佐藤さんの技術力と、お客様のニーズ。この両方を組み合わせれば、最強です」

二人は固く握手を交わした。

## エピローグ：マーケットインの本質

その夜、山田は自室で今日の会議を振り返っていた。

マーケットインとは何か。それは単なる手法ではない。思想であり、哲学だ。

「お客様のために」という言葉は簡単だ。しかし、本当に実践するのは難しい。自分たちの技術、自分たちの都合、自分たちの論理。それらを一旦脇に置いて、純粋にお客様の立場で考える。

そして、お客様すら気づいていない本質的なニーズを見抜く。そのニーズに対して、最高の解決策を提供する。技術はそのための手段だ。

山田はノートに書き留めた。

```
【マーケットインの三原則】

1. 表面的な声ではなく、本質的ニーズを見抜く
2. 技術は手段、目的は顧客の課題解決
3. 「世界初」は結果であり、目的ではない
```

真のイノベーションは、技術の独りよがりからは生まれない。お客様のニーズを深く理解し、そこから逆算して考える。その先に、誰も見たことのない価値が生まれる。

山田は窓の外を見た。街には無数の人々が暮らしている。それぞれが課題を抱え、解決策を求めている。

その声に耳を傾け、本質を見抜き、最適な解決策を提供する。それこそが、ビジネスの原点だ。

明日から、新しいプロジェクトが始まる。お客様に寄り添う、新しい価値を創造するプロジェクトが。

山田は決意を新たに、明日への準備を始めた。

## 第18章のポイント

### プロダクトアウト vs マーケットイン

**プロダクトアウト**
- 技術・製品起点の発想
- 「この技術が素晴らしい」から始まる
- 独りよがりになるリスク

**マーケットイン**
- 顧客ニーズ起点の発想
- 「お客様の課題は何か」から始まる
- 本質的な価値創造につながる

### イノベーションの本質

1. 表面的な声ではなく、本質的ニーズを見抜く
2. 「なぜ？」を5回繰り返して深掘りする
3. 技術は手段、目的は顧客の幸せ
4. 「世界初」は結果であり、目的ではない

### 実践のポイント

- 顧客との対話を重視する
- 言葉の裏にある真意を読み解く
- 技術者も顧客の声を直接聞く
- ビジネスモデルも含めて最適化する

**次章予告：第19章「組織の壁を越える」**

新プロジェクトが始動したが、部門間の壁が立ちはだかる。技術部門、営業部門、企画部門。それぞれの論理がぶつかり合う中、山田はどう組織を一つにまとめていくのか。組織マネジメントの本質が明らかになる。
