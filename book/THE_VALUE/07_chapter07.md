# 第7章 顧客を「完全に」理解するとは

## 明確に・完全に・認識のズレなく

---

## 1. 再訪問の朝

翌週の水曜日、田中は朝一番でナカムラ工業を再訪していた。

「おはようございます、田中さん」

中村社長は以前とは異なり、どこか期待に満ちた表情で迎えてくれた。

「前回、私たちの課題を丁寧に聞いてくださいましたよね。正直、あんなに時間をかけて話を聞いてくれたベンダーは初めてでした」

田中は軽く頷いた。

「ありがとうございます。ただ、前回はまだ『聞いた』だけです。今日は、お客様の課題を『完全に理解する』ために伺いました」

「完全に？」

中村社長が首を傾げる。

「はい。私たちには『ニーズの3段階理解』という考え方があります。今日はそれを実践させてください」

田中はそう言うと、分厚いヒアリングシートをテーブルに広げた。

---

## 2. 第1段階：明確に理解する

「まず第1段階は『明確に理解する』です。これはWhat（具体的に何か）とWhy（なぜそれが重要か）を明らかにすることです」

田中はシートの最初のページを開いた。

**【課題1：在庫管理の非効率】**

「前回、在庫管理が非効率だとおっしゃいましたね。具体的に、どの工程で、どのような問題が起きているのか教えていただけますか？」

中村社長は少し考えてから答えた。

「うちは部品が3,000アイテムあるんですが、それを手書きの台帳で管理しているんです。入庫・出庫のたびに記入して、月末に棚卸しをする。でも、記入漏れや誤記が多くて...」

「記入漏れや誤記は、月に何件くらい発生しますか？」

「そうですね...平均で月に50件くらいでしょうか」

田中はメモを取りながら続けた。

「50件の記入ミスによって、具体的にどんな問題が起きていますか？」

「まず、棚卸しに時間がかかります。実在庫と帳簿が合わないので、一つ一つ確認しないといけない。それに、欠品が起きることもあります。帳簿上は在庫があるはずなのに、実際には無い、とか」

「欠品による機会損失は、金額にするとどのくらいですか？」

中村社長は少し驚いた表情を見せた。

「そこまで計算したことはなかったですが...月に2〜3回欠品が起きて、平均すると1回あたり20万円くらいの受注を逃していると思います」

「つまり、月間で約60万円、年間で720万円の機会損失ということですね」

田中は数字を書き込んだ。

「そうか...そんなに損失が出ていたのか」

中村社長は深く息を吐いた。

「これが『明確に理解する』ということです。『在庫管理が非効率』という抽象的な課題を、『月50件の記入ミス、年間720万円の機会損失』という具体的な数字に落とし込む。そして、なぜそれが重要なのか—御社の収益に直接影響するから—という理由まで明らかにする」

中村社長は何度も頷いた。

---

## 3. 第2段階：完全に理解する

「では、第2段階に進みます。『完全に理解する』です」

田中は新しいページを開いた。

「これはMECE（Mutually Exclusive, Collectively Exhaustive）、つまり『漏れなく・ダブりなく』すべての課題を洗い出し、さらに優先順位も含めて理解するということです」

田中はホワイトボードに図を描き始めた。

```
【ナカムラ工業の課題全体像】

1. 在庫管理の非効率
   ├─ 手書き台帳による記入ミス（月50件）
   ├─ 棚卸し作業の長時間化（月末3日間）
   └─ 欠品による機会損失（年間720万円）

2. 製造工程の情報共有不足
   ├─ 進捗状況の可視化不足
   ├─ 部門間の情報伝達遅延
   └─ 納期遅延（月2〜3件）

3. 顧客情報の属人化
   ├─ 営業担当者のみが顧客情報を把握
   ├─ 引き継ぎ時の情報欠損
   └─ リピート率の低下

4. 経営数値の把握遅延
   ├─ 月次決算に2週間
   ├─ リアルタイムな意思決定困難
   └─ 機会損失の拡大
```

「前回のヒアリングから、大きく分けて4つの課題領域があることが分かりました。ただ、すべてを同時に解決することはできません。優先順位をつける必要があります」

田中は中村社長を見た。

「社長、この中で最も解決したい課題はどれですか？ そして、その理由は？」

中村社長はしばらく図を見つめていた。

「やはり...在庫管理ですかね。年間720万円の損失は大きいですし、何より従業員が月末の棚卸しで疲弊しているのを見るのが辛いんです」

「分かりました。では、次に重要なのは？」

「製造工程の情報共有です。納期遅延はお客様の信頼を失いますから」

田中は優先順位を番号で振っていった。

「完璧です。これで『完全に理解する』ができました。すべての課題を漏れなく洗い出し、それぞれの影響度を明確にし、優先順位まで決めた。これが第2段階です」

---

## 4. 第3段階：認識のズレなく理解する

「そして最後、第3段階が最も重要です」

田中は真剣な表情になった。

「それは『認識のズレなく理解する』—お客様の頭の中と同じ絵を描けるレベルまで理解することです」

田中はノートを取り出し、手描きで図を描き始めた。

「社長、今から私が描く絵を見てください。これが私の理解している『ナカムラ工業の理想の姿』です」

```
【ナカムラ工業の6ヶ月後の姿】

朝8時
├─ 全従業員がタブレットで在庫状況を確認
├─ 前日の出庫・入庫が自動で反映されている
└─ 欠品リスクのある部品は赤色でアラート表示

製造中
├─ 各工程の進捗がリアルタイムで可視化
├─ 遅延が発生したら即座に関係部署に通知
└─ 営業担当が顧客に正確な納期を即答できる

月末
├─ 棚卸し作業が半日で完了（従来3日→0.5日）
├─ 在庫差異が月10件以下（従来50件→10件）
└─ 従業員が定時で帰宅できる

経営会議
├─ 社長がリアルタイムで経営数値を把握
├─ 在庫回転率、粗利率などのKPIをダッシュボード表示
└─ データに基づいた意思決定が可能に
```

田中は絵を描き終えると、中村社長に見せた。

「社長の頭の中にある『理想の姿』は、これと同じですか？ それとも違いますか？」

中村社長は絵をじっと見つめた。

長い沈黙の後、彼は口を開いた。

「ほとんど合っています。でも...一つだけ違うかもしれません」

「どこでしょうか？」

「『従業員が定時で帰宅できる』という部分。私が本当に実現したいのは、『従業員が家族との時間を増やせる』ことなんです」

中村社長の目が少し潤んだ。

「うちの従業員は皆、家族を大切にしています。でも、月末の棚卸しのせいで、子供の運動会に行けなかったり、家族旅行をキャンセルしたりしている。それが...本当に申し訳なくて」

田中は静かに頷き、絵に書き加えた。

```
月末
├─ 棚卸し作業が半日で完了
├─ 在庫差異が月10件以下
└─ 従業員が家族との時間を増やせる
   （子供の行事参加、家族旅行など）
```

「これで完璧ですか？」

「はい...これです。これが私の描いている理想の姿です」

中村社長は深く頷いた。

---

## 5. 同じ絵を描く

田中はゆっくりと立ち上がった。

「社長、今、私たちは『同じ絵』を見ています」

彼は描いた図を指差した。

「これが『認識のズレなく理解する』ということです。単に課題を聞くだけではなく、お客様が本当に実現したい未来、その背景にある想い、そこに込められた価値観まで—すべてを共有する」

中村社長は黙って聞いていた。

「私たちシステム開発会社は、よく『要件定義』という言葉を使います。でも、本当に大切なのは『要件』ではなく『想い』なんです。社長が従業員の家族を想う気持ち、それを実現するための手段としてシステムがある」

田中は中村社長の目を見た。

「今日のヒアリングで、私は御社の課題を『明確に』理解しました。すべての課題を『完全に』洗い出しました。そして何より、社長の想いを『認識のズレなく』理解できました」

「田中さん...」

中村社長は言葉に詰まった。

「ありがとうございます。正直、これまで何社かのシステム会社と話をしてきましたが、ここまで私たちのことを理解しようとしてくれた人はいませんでした」

「それは、彼らが悪いわけではありません。ただ、『聞く技術』を学んでいなかっただけです」

田中はヒアリングシートを閉じた。

「私の師匠は、こう言いました。『顧客理解には3つの段階がある。明確に理解すること、完全に理解すること、そして認識のズレなく理解すること。この3つができて初めて、本当の価値を提供できる』と」

---

## 6. 信頼の深化

その日の帰り道、田中は林部長に報告の電話を入れた。

「部長、ナカムラ工業のヒアリングが完了しました」

「そうか。どうだった？」

「3段階理解のフレームワークを実践しました。特に第3段階—認識のズレなく理解する—が効果的でした。中村社長の本当の想いを引き出すことができました」

林部長は満足そうに答えた。

「良かった。で、その『想い』とは何だった？」

「従業員が家族との時間を増やせるようにしたい、ということです」

「...なるほど」

林部長は少し間を置いてから続けた。

「田中、それが『本当のニーズ』だ。在庫管理の効率化は手段であって、目的ではない。中村社長の本当の目的は、従業員の幸せなんだ」

「はい。それが分かったとき、私も社長も『同じ絵』を見ている感覚がありました」

「それだよ、田中。それが顧客を『完全に』理解するということだ」

---

## 7. 翌日のオフィス

翌日、田中は新人の佐藤に声をかけた。

「佐藤さん、昨日のナカムラ工業のヒアリング、どうだった？」

佐藤は同行していた新人だった。

「すごく勉強になりました。特に『認識のズレなく理解する』という部分。あの、社長が『家族との時間』と言ったとき、田中さんがすぐに図に書き加えたじゃないですか。あれが印象的で」

「あれが重要なんだ。お客様の言葉を『そのまま』受け止め、『そのまま』形にする。自分の解釈を挟まない」

田中はホワイトボードに書いた。

```
【顧客理解の3段階】

第1段階：明確に理解する
├─ What（具体的に何か）
└─ Why（なぜ重要か）

第2段階：完全に理解する
├─ MECE（漏れなく・ダブりなく）
└─ Priority（優先順位）

第3段階：認識のズレなく理解する
├─ 顧客の頭の中と同じ絵を描く
├─ 言葉の裏にある想いを捉える
└─ 自分の解釈を挟まない
```

「この3段階を踏むことで、初めて本当の価値提供ができる。逆に言えば、どれか一つでも欠けると、顧客の期待と違うものを作ってしまう」

佐藤は真剣にメモを取っていた。

「田中さん、質問があります」

「何？」

「第3段階の『認識のズレなく理解する』って、どうやったらできるようになりますか？」

田中は少し考えてから答えた。

「それはね...『聞く』だけじゃなくて『描く』ことだよ」

「描く？」

「そう。お客様の言葉を聞いたら、それを絵や図にして見せる。『私の理解はこれで合っていますか？』と確認する。そうすると、お客様も『ああ、ここが違う』『これが抜けている』と教えてくれる」

佐藤は目を輝かせた。

「なるほど...言葉だけだと、どうしても解釈のズレが生まれるんですね」

「その通り。だから私たちは、常に『可視化』する。図を描く、表を作る、プロトタイプを見せる。それが『同じ絵を描く』ための最も確実な方法だ」

---

## 8. 本質の理解

その日の夕方、林部長が田中のデスクに立ち寄った。

「田中、ナカムラ工業の提案書、見させてもらったよ」

「ありがとうございます」

「良くできている。特に『従業員の家族との時間創出』を最上位の目的に置いた構成が素晴らしい」

林部長は資料をめくりながら続けた。

「多くのSEは『在庫管理システムの導入』で終わらせてしまう。でも、お客様が本当に求めているのはシステムじゃない。システムの先にある『価値』なんだ」

「はい。それを実感しました」

「田中、覚えておいてほしい。顧客を『完全に』理解するということは、顧客が言葉にしていないことまで理解するということだ」

田中は頷いた。

「中村社長は『在庫管理を効率化したい』と言った。でも、その奥には『従業員を大切にしたい』という想いがあった。その想いを引き出し、共有できたからこそ、本当の提案ができた」

「そういうことだ」

林部長は満足そうに笑った。

「次は、その提案をどう形にするか、だな」

「はい。次回の訪問で、プロトタイプをお見せする予定です」

「楽しみにしているよ」

---

## 9. 1週間後

1週間後、田中は再びナカムラ工業を訪れた。

今回は、簡単なプロトタイプを持参していた。

「社長、前回のヒアリングを元に、簡単な画面イメージを作ってきました」

タブレットに映し出されたのは、シンプルな在庫管理画面だった。

```
【在庫管理ダッシュボード】

今日の状況
├─ 欠品リスク：部品A、部品C（赤色表示）
├─ 発注推奨：部品B、部品D（黄色表示）
└─ 在庫適正：その他2,996アイテム（緑色表示）

今月の実績
├─ 棚卸し差異：8件（目標10件以下）
├─ 欠品件数：0件（前月比-3件）
└─ 在庫回転率：4.2回（前月比+0.3）

従業員の声
├─ 「月末が楽になりました」（製造部・山田さん）
├─ 「子供の運動会に行けました」（倉庫・鈴木さん）
└─ 「家族旅行を計画できました」（営業・田中さん）
```

中村社長は画面を食い入るように見つめた。

「これは...」

「はい。前回、社長が描いてくださった『理想の姿』を形にしてみました」

特に最後の『従業員の声』の部分で、中村社長の目が潤んだ。

「田中さん、これです。これが私の見たかった景色です」

「ありがとうございます。これはまだプロトタイプですが、私たちが作るシステムの最終的な姿を表しています」

中村社長は何度も頷いた。

「分かりました。御社にお願いします」

---

## 10. その夜のリフレクション

その日の夜、田中はノートに書き留めた。

```
【今日の学び：顧客を完全に理解するとは】

1. 明確に理解する
   - 抽象的な課題を具体的な数字に落とす
   - What（何が）とWhy（なぜ）を明確にする

2. 完全に理解する
   - MECEですべての課題を洗い出す
   - 優先順位まで含めて理解する

3. 認識のズレなく理解する
   - 顧客の頭の中と同じ絵を描く
   - 言葉の裏にある想いを捉える
   - 可視化して確認する

そして最も重要なこと：
顧客が本当に求めているのは「システム」ではなく、
その先にある「価値」である。

中村社長の場合、それは「従業員の幸せ」だった。
```

田中は窓の外を見た。

夜空には満月が輝いていた。

「顧客を完全に理解する—それは、顧客と同じ未来を見ることだ」

彼はそう呟くと、次の提案資料の作成に取りかかった。

---

**第7章 完**

---

## 本章のキーメッセージ

1. **明確に理解する**：抽象的な課題を具体的な数字とWhatWhy に落とし込む
2. **完全に理解する**：MECEですべてを洗い出し、優先順位まで把握する
3. **認識のズレなく理解する**：顧客の頭の中と同じ絵を描き、想いまで共有する
4. **可視化の重要性**：言葉だけでなく、図や画面で「見せる」ことで認識を合わせる
5. **本質の追求**：顧客が求めるのはシステムではなく、その先にある「価値」

---

次章予告：第8章「価値を『正確に』定義する」では、理解した顧客ニーズを、どのように提案書や要件定義書に落とし込むかを描きます。「価値の言語化」の技術を学びます。
