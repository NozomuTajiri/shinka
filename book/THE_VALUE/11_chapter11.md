# 第11章「伝わらなければ、価値はゼロ」

お客様の言葉で、お客様の目線で

---

## 1. 新しい提案書の準備

ナカムラ工業での成功から一週間が経った月曜日の朝、山田は佐藤課長に声をかけた。

「佐藤、次の案件の提案書、見せてくれるか」

佐藤が手渡した資料には、「カワシマ精密工業向け提案書」というタイトルがあった。カワシマ精密工業は、自動車部品メーカーで、検査工程の自動化に興味を示していた。ナカムラ工業と同じようなニーズを持つ顧客だ。

山田は資料をめくり始めた。

一ページ目から、違和感を覚えた。

```
【提案概要】
本提案は、当社の次世代型検査自動化ソリューション
「SmartInspect Pro X2000」の導入により、
貴社における品質管理業務の抜本的な改革を実現するものです。

【技術仕様】
- センサー精度：0.001mm
- 処理速度：30個/分
- インターフェース：RS-485、Modbus TCP対応
- 電源：AC100-240V、50/60Hz対応
```

山田は、ページを進めるごとに眉間の皺が深くなっていった。

五ページまで読んだところで、山田は資料を閉じた。

「佐藤、この提案書、誰に向けて書いたんだ？」

「え？ カワシマ精密工業の購買部長、斉藤さんですが……」

「斉藤さんは、技術者か？」

佐藤は首を横に振った。

「いえ、購買畑の方です」

「じゃあ、この『Modbus TCP』とか『RS-485』とか、分かると思うか？」

佐藤の表情が曇った。

「それは……専門用語なので、分からないかもしれませんが、技術的な信頼性を示すために……」

山田は深く息を吐いた。

「佐藤、これじゃダメだ」

---

## 2. 宮本の指摘

その日の午後、山田は宮本のもとを訪れた。

佐藤が作成した提案書を持って。

「先生、相談があります」

宮本は珈琲を淹れながら、山田の話を聞いた。

「部下が作った提案書なんですが、どうも……伝わらない気がするんです」

「見せてくれ」

宮本は提案書をゆっくりとめくった。

一ページ、二ページ、三ページ。

そして、資料を閉じた。

「山田君、この提案書、誰に向けて書いた？」

「カワシマ精密工業の購買部長です」

「では、この購買部長は、どんな人だ？」

山田は少し考えた。

「五十代半ば。購買畑一筋で、技術的な知識はそれほど深くないと聞いています。ただ、コスト意識は非常に高い方です」

「そうか」宮本は頷いた。「では、この購買部長が、この提案書を読んで、何を感じると思う？」

山田は答えに窮した。

「それは……」

「分からない、だろう？」宮本は優しく笑った。「なぜなら、この提案書は『誰のために書かれたのか』が曖昧だからだ」

宮本は、ホワイトボードに大きく書いた。

```
【提案書の致命的な3つの罪】

1. 誰に向けて書いているのか不明確
2. 専門用語だらけで相手の言葉になっていない
3. 相手が本当に知りたいことが書かれていない
```

「山田君、良い提案書とは何だと思う？」

山田は少し考えてから答えた。

「技術的に正確で、詳細な仕様が書かれていて……」

「違う」宮本がきっぱりと言った。「良い提案書とは、『読んだ人が、次の行動を起こしたくなる』提案書だ」

---

## 3. Before：伝わらない提案書

宮本は、佐藤の提案書を開いた。

「では、この提案書の何が問題か、一緒に見ていこう」

宮本が指さしたのは、冒頭の文章だった。

```
【Before】
本提案は、当社の次世代型検査自動化ソリューション
「SmartInspect Pro X2000」の導入により、
貴社における品質管理業務の抜本的な改革を実現するものです。
```

「これを読んで、山田君は何を感じる？」

「……専門的で、しっかりした印象ですが」

「では、カワシマ精密工業の購買部長は、何を感じる？」

山田は想像してみた。

五十代の購買部長が、この文章を読む。

「……よく分からない、かもしれません」

「その通り」宮本が頷いた。「『次世代型検査自動化ソリューション』『品質管理業務の抜本的な改革』—どれも抽象的で、何が起きるのかイメージできない」

宮本は、次のページを指さした。

```
【Before】
【技術仕様】
- センサー精度：0.001mm
- 処理速度：30個/分
- インターフェース：RS-485、Modbus TCP対応
- 電源：AC100-240V、50/60Hz対応
```

「これは？」

「技術仕様ですね。正確な情報だと思いますが」

「正確かもしれない。でも、購買部長にとって、この情報は意味があるか？」

山田は考えた。

購買部長が知りたいのは、技術仕様ではない。

「コストがどれくらい削減できるか、だと思います」

「そうだ」宮本は力強く言った。「相手が知りたいことと、こちらが伝えたいことが、ズレている。これが、伝わらない提案書の最大の問題だ」

---

## 4. お客様の言葉で語る

宮本は、新しいノートを開いた。

「では、どうすれば伝わる提案書になるか。最も重要な原則を教えよう」

宮本は大きく書いた。

```
【原則1】
お客様の言葉で語る
```

「お客様の言葉……？」

「そうだ。専門用語ではなく、お客様が日常的に使っている言葉で書く。技術者向けなら技術用語でもいい。でも、購買部長向けなら、購買部長が使う言葉で書く」

宮本は、先ほどの文章を書き直し始めた。

```
【After】
御社の検査工程では、現在5人の検査員が
1日8時間かけて全数検査を行っています。

私たちの提案するシステムを導入すると、
検査時間が40%短縮され、
検査員を3人に減らすことができます。

つまり、年間で約1,200万円の
人件費削減が可能になります。
```

「どうだ？」

山田は、二つの文章を見比べた。

Beforeの文章は、確かに専門的で立派に見える。しかし、何が起きるのか、具体的にイメージできない。

Afterの文章は、シンプルだ。しかし、何が起きて、何が得られるのかが、はっきりと分かる。

「これなら……購買部長も理解できますね」

「そうだ。そして、もっと重要なことがある」

宮本は続けた。

「この『年間1,200万円の削減』という数字は、どこから来た？」

山田は気づいた。

「カワシマ精密工業の、現状の検査体制から計算したんですね」

「その通り。つまり、これは『一般論』ではなく、『カワシマ精密工業だけの、具体的な数字』だ。それが伝わるから、説得力がある」

---

## 5. お客様の目線で語る

宮本は、次の原則を書いた。

```
【原則2】
お客様の目線で語る
```

「技術仕様のページを見てくれ」

宮本が指さしたのは、先ほどの技術仕様だった。

```
【Before】
【技術仕様】
- センサー精度：0.001mm
- 処理速度：30個/分
- インターフェース：RS-485、Modbus TCP対応
- 電源：AC100-240V、50/60Hz対応
```

「この仕様、何が問題だと思う？」

「……お客様の目線になっていない、ということですか？」

「そうだ。これは『製品の視点』で書かれている。でも、お客様が知りたいのは『製品の仕様』じゃない。『自分たちにとって、どんな良いことがあるのか』だ」

宮本は、書き直した。

```
【After】
【導入後の変化】

検査精度が向上します
└─ 現在の目視検査では見逃していた
   0.01mm以下の微細な傷も検出できます
   → 不良品の市場流出リスクがゼロになります

検査スピードが上がります
└─ 1個30秒かかっていた検査が、
   2秒で完了します
   → 1日の生産量が1.5倍に増えます

既存の設備にすぐ接続できます
└─ 御社が現在お使いの検査機と
   接続するだけで使えます
   → 新しい設備投資は不要です
```

山田は、息を飲んだ。

同じ情報なのに、伝わり方がまったく違う。

Beforeは、「何ができるか」を語っている。

Afterは、「お客様にとって、どんな良いことがあるか」を語っている。

「これが、『お客様の目線で語る』ということだ」宮本が説明した。「『0.001mm』という仕様そのものは、お客様にとってはどうでもいい。でも、『不良品の市場流出リスクがゼロになる』なら、意味がある」

---

## 6. 書き直しの実践

「では、山田君、実際にやってみよう」

宮本は、提案書の別のページを開いた。

```
【Before】
【導入効果】
- 品質管理業務の効率化
- トレーサビリティの向上
- データドリブンな意思決定の実現
- コンプライアンス対応の強化
```

「これを、お客様の言葉と目線で書き直してみてくれ」

山田は、ペンを取った。

カワシマ精密工業の購買部長、斉藤さんの顔を思い浮かべた。

五十代半ば。コスト意識が高い。技術的な詳細よりも、ビジネスへの影響を知りたい人だ。

斉藤さんなら、何を知りたいだろう。

どんな言葉を使うだろう。

山田は、書き始めた。

```
【After】
【このシステムを導入すると】

月末の棚卸し時間が3日から半日になります
└─ 検査員5人が、月末に徹夜していた作業が
   半日で終わるようになります
   → 従業員が家族との時間を持てるようになります

不良品による損失がなくなります
└─ これまで年間20件発生していた
   クレーム対応コスト（平均50万円/件）が
   ゼロになります
   → 年間1,000万円の損失を防げます

データに基づいた改善ができます
└─ どの工程で、いつ不良が発生しているか、
   すべてデータで見えるようになります
   → 勘ではなく、事実に基づいた改善ができます
```

山田が書き終えると、宮本は目を通した。

「素晴らしい」

宮本は満足そうに頷いた。

「特に良いのは、『従業員が家族との時間を持てる』という部分だ。これは、コスト削減という数字以上に、経営者の心に響く」

「ありがとうございます」

「ただ、一つだけ」

宮本が指摘した。

「最後の『データに基づいた改善』の部分。『勘ではなく、事実に基づいた』というのは良いが、もう少し具体的に書けないか？」

山田は考えた。

「例えば……こうでしょうか」

```
データに基づいた改善ができます
└─ 「午後3時以降に不良品が増える」
   「特定のロットで精度が落ちる」
   こうした傾向がデータで見えるようになります
   → 作業時間の見直しや、材料の変更など、
      具体的な改善策を打てるようになります
```

「完璧だ」宮本が拍手した。「具体例があると、イメージが一気に湧く。これが『伝わる』提案書だ」

---

## 7. 最大の違い：誰のための提案書か

宮本は、コーヒーを飲み終えると、最後の原則を書いた。

```
【原則3】
誰のための提案書かを、常に意識する
```

「山田君、提案書を書くとき、多くの営業マンが陥る罠がある」

「罠、ですか？」

「そうだ。それは、『自分が言いたいことを書く』という罠だ」

宮本は真剣な顔で続けた。

「提案書は、自分のためのものじゃない。お客様のためのものだ。だから、『自分が伝えたいこと』ではなく、『お客様が知りたいこと』を書く」

山田は深く頷いた。

「今回の提案書も、そうだったかもしれません。『うちの製品はこんなに凄いんです』と言いたかった」

「その気持ちは分かる。でも、お客様にとって、君の製品が凄いかどうかは、どうでもいい」

「……」

「お客様が知りたいのは、ただ一つ。『自分たちの課題が、解決できるのか』だ」

宮本は、Before とAfterの提案書を並べた。

```
【Before】
当社の次世代型検査自動化ソリューション
「SmartInspect Pro X2000」は……

【After】
御社の検査工程の課題—
「検査に時間がかかりすぎる」
「人手が足りない」
「不良品を見逃してしまう」
これらを解決できます
```

「見てくれ。Beforeは『当社の製品』から始まる。Afterは『御社の課題』から始まる。この違いが、すべてだ」

山田は、二つの文章を見比べた。

確かに、Afterの方が、自分事として読める。

「お客様の課題から始める。そして、その課題を解決する方法として、自社の製品を紹介する。この順番が重要なんです」

「その通り」宮本が力強く言った。「主役は、お客様だ。製品じゃない」

---

## 8. 佐藤への添削

翌日、山田は佐藤を呼んだ。

「佐藤、提案書、一緒に直そう」

二人は、会議室にこもった。

ホワイトボードに、Before と After を並べて書いた。

「まず、冒頭から変えよう」

山田は、佐藤と一緒に書き直し始めた。

```
【Before】
本提案は、当社の次世代型検査自動化ソリューション
「SmartInspect Pro X2000」の導入により、
貴社における品質管理業務の抜本的な改革を実現するものです。

【After】
カワシマ精密工業様へ

先日の打ち合わせで、斉藤部長からお聞きした課題：

「検査に人手がかかりすぎて、他の業務に手が回らない」
「月末の棚卸しで従業員が疲弊している」
「不良品の見逃しが心配で、夜も眠れない」

これらの課題を解決する方法を、ご提案します。
```

佐藤は、二つの文章を見比べた。

「確かに……Afterの方が、読んだ瞬間に『自分のことだ』と思いますね」

「そうだ。提案書の最初の5行で、読むか読まないかが決まる」山田が説明した。「だから、最初に『あなたのことを理解していますよ』と伝える」

二人は、一ページずつ、丁寧に書き直していった。

専門用語を、お客様の言葉に置き換えた。

技術仕様を、お客様にとっての価値に翻訳した。

自社の視点を、お客様の視点に変えた。

三時間後、新しい提案書が完成した。

「どうだ？」

山田が聞くと、佐藤は感動した表情で答えた。

「まったく別の資料になりました。こっちの方が、絶対に伝わります」

---

## 9. 提案の結果

一週間後、山田と佐藤はカワシマ精密工業を訪れた。

購買部長の斉藤は、新しい提案書を真剣に読んでいた。

五分後、斉藤が顔を上げた。

「山田さん、この提案書、素晴らしいですね」

「ありがとうございます」

「実は、他社からも提案をもらっているんですが……」斉藤が別の資料を取り出した。「こちらは、技術的な仕様ばかりで、正直何が良いのかよく分からなかった」

その資料を見ると、まさにBeforeの提案書と同じスタイルだった。

「でも、御社の提案書は違う。『私たちの課題』が最初に書いてある。『何が解決できるのか』が明確に書いてある。そして何より、『従業員が家族との時間を持てる』という部分に、心が動きました」

斉藤は、資料を閉じた。

「うちの社長も読んだんですが、『この会社は、本当にうちのことを考えてくれている』と言っていました」

山田は、宮本の言葉を思い出した。

『良い提案書とは、読んだ人が次の行動を起こしたくなる提案書だ』

「山田さん、御社の提案を採用させていただきます」

---

## 10. 伝わる喜び

その日の夜、山田は宮本に報告の電話をした。

「先生、カワシマ精密工業から受注をいただきました」

「それは良かった」

「提案書を書き直したことが、決め手になりました。斉藤部長が『本当にうちのことを考えてくれている』と言ってくださって」

「それが、伝わるということだ」宮本が電話の向こうで言った。「どんなに良い価値も、伝わらなければ意味がない。逆に、きちんと伝われば、お客様は必ず応えてくれる」

山田は、深く頷いた。

「今回、本当に学びました。提案書は、自分のためじゃない。お客様のために書くものだと」

「そうだ。そして、もう一つ」

「はい」

「伝わる提案書を書くには、まず『徹底的に顧客を理解する』ことが前提だ。カワシマ精密工業の課題は何か、購買部長がどんな言葉を使うか、何を大切にしているか。それが分かっているから、伝わる言葉で書ける」

山田は、この数週間の学びを振り返った。

宮本から学んだのは、営業の技術だけではなかった。

「お客様を理解する」
「お客様の言葉で語る」
「お客様の目線で考える」

それは、すべて『相手への敬意』から始まっていた。

「先生、ありがとうございます」

「まだ道は長いぞ」宮本が笑った。「次は、この『伝わる提案書』を、部下たち全員が書けるようにすることだ」

「はい。必ずやります」

---

## 11. 営業部への展開

翌週の月曜日、山田は営業部全員を集めた。

「今日から、提案書の書き方を変える」

ホワイトボードに、大きく書いた。

```
【伝わる提案書の3原則】

1. お客様の言葉で語る
   └─ 専門用語ではなく、相手が日常使う言葉で

2. お客様の目線で語る
   └─ 「製品の凄さ」ではなく「お客様の得られる価値」を

3. 誰のための提案書かを意識する
   └─ 主役はお客様、製品は脇役
```

「この3つを守れば、提案書の質が劇的に変わる」

若手社員が手を挙げた。

「部長、でも専門用語を使わないと、技術的な信頼性が下がりませんか？」

「良い質問だ」山田が答えた。「専門用語が必要な場合もある。でも、それは『相手が技術者の場合』だ。購買部長や経営者に対して専門用語を使うのは、『伝えたいから』じゃなく、『自分が楽だから』だ」

会議室がシンと静まった。

「お客様の言葉で語るのは、確かに難しい。相手がどんな言葉を使うのか、何を大切にしているのか、理解していないとできない」

山田は続けた。

「でも、だからこそ価値がある。『この営業は、自分のことを本当に理解してくれている』と思ってもらえる」

佐藤課長が立ち上がった。

「部長、実は僕も最初の提案書では同じ間違いをしていました。でも、部長と一緒に書き直して、カワシマ精密工業から『こんなに分かりやすい提案書は初めてだ』と言っていただけました」

佐藤は、Before と After の提案書を配った。

「見てください。同じ内容なのに、伝わり方がまったく違います」

営業部のメンバーたちは、二つの提案書を真剣に読み比べた。

「本当だ……Afterの方が、圧倒的に読みやすい」

「これなら、技術に詳しくない人でも理解できる」

山田は満足そうに頷いた。

「今日から、全員の提案書を、この3原則でチェックする。最初は大変かもしれないが、必ず結果に繋がる」

## 本章のキーメッセージ

**伝わらなければ、価値はゼロ**

どんなに優れた価値を提供できても、それが相手に伝わらなければ意味がない。

伝わる提案書の3原則：
1. **お客様の言葉で語る** - 専門用語ではなく、相手が日常的に使う言葉で
2. **お客様の目線で語る** - 製品の凄さではなく、お客様が得られる価値を
3. **誰のための提案書かを意識する** - 主役はお客様、製品は脇役

Before/Afterの具体例：
- Before: 「次世代型検査自動化ソリューションの導入により……」
- After: 「御社の検査工程の課題を解決できます」

そして何より重要なのは、伝わる提案書を書くには、まず「徹底的に顧客を理解する」ことが前提だということ。

相手を理解し、相手の言葉で、相手の目線で語る。

それが、真のコミュニケーションである。
