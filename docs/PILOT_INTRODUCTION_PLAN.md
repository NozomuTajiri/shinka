# カクシン既存顧客パイロット導入計画書
## Miyabi AI自律開発プラットフォーム パイロットプログラム

**作成日**: 2024年11月30日
**バージョン**: 1.0.0
**対象**: 株式会社カクシン既存顧客
**プロジェクト期間**: 3ヶ月（2025年Q1-Q2想定）
**親Issue**: #2 カクシン進化

---

## エグゼクティブサマリー

本計画書は、株式会社カクシンの既存顧客（500社以上のコンサルティング実績）を対象とした、Miyabi AI自律開発プラットフォームのパイロット導入プログラムを定義します。

### 目的

1. **プロダクトマーケットフィット（PMF）検証**: エンタープライズ市場での実用性確認
2. **ケーススタディ作成**: 導入事例として活用可能な定量的成果の獲得
3. **フィードバック収集**: プロダクト改善のための実践的知見獲得
4. **初期顧客獲得**: 有償契約への転換（目標: 20社中10社以上）
5. **推薦状取得**: 営業資産としての推薦状・ロゴ使用許諾

### 期待成果

| 成果指標 | 目標値 |
|---------|--------|
| **パイロット参加企業** | 20社 |
| **有償転換率** | 50%以上（10社以上） |
| **開発生産性向上** | 平均5倍以上 |
| **顧客満足度（CSAT）** | 80%以上 |
| **推薦状取得** | 15社以上 |
| **ケーススタディ** | 5業界 × 各1社 = 5本 |

## 1. パイロット顧客選定基準

### 1.1 選定条件（必須）

| 基準 | 条件 | 理由 |
|------|------|------|
| **カクシン実績** | コンサルティング実績あり | 信頼関係構築済み、意思決定が早い |
| **組織規模** | 従業員500名以上 | エンタープライズ顧客の代表性 |
| **IT投資意欲** | 年間IT投資1億円以上 | 予算確保の実現可能性 |
| **開発部門** | 社内エンジニア10名以上 | 導入効果の定量測定が可能 |
| **DX推進** | DX推進部門が存在 | 新技術導入の組織的受容性 |
| **経営層コミット** | CIO/CTO級の賛同 | トップダウンでの導入推進 |

### 1.2 選定条件（優先）

| 基準 | 条件 | 配点 |
|------|------|------|
| **業界多様性** | 異なる業界（製造、IT、人材、金融、サービス） | +20点 |
| **DX成熟度** | 高（既にクラウド活用、アジャイル開発導入） | +15点 |
| **ブランド価値** | 上場企業、または業界リーディングカンパニー | +15点 |
| **地理的多様性** | 本社が東京以外（大阪、名古屋等） | +10点 |
| **導入体制** | 専任PM配置可能 | +10点 |
| **フィードバック意欲** | 週次MTG参加可能 | +10点 |
| **ケーススタディ協力** | 社名公開・推薦状提供OK | +20点 |

**合計点**: 100点満点
**選定基準**: 必須条件をすべて満たし、優先条件で60点以上

### 1.3 選定プロセス

```
Step 1: カクシン既存顧客リスト抽出（50社）
    ↓
Step 2: 必須条件によるスクリーニング（→ 30社）
    ↓
Step 3: 優先条件スコアリング（60点以上 → 25社）
    ↓
Step 4: 初回アプローチ・ヒアリング（興味あり → 20社）
    ↓
Step 5: パイロット参加正式決定（20社）
```

**スケジュール**: 2週間

4. どのような機能があれば、さらに活用できますか？
5. 有償契約への転換を検討されますか？（Yes/No、理由）

### 5.3 フィードバック管理

- **ツール**: Notion、GitHub Issues（Feature Request Label）
- **優先度付け**: Impact（影響度）× Feasibility（実現可能性）でスコアリング
- **対応方針**:
  - P0（緊急）: 24時間以内対応
  - P1（高）: 1週間以内対応
  - P2（中）: 次回リリースで対応
  - P3（低）: バックログ登録

## 7. リスクと対策

### 7.1 主要リスク

| リスク | 発生確率 | 影響度 | 対策 |
|--------|---------|--------|------|
| **参加企業の離脱** | 中 | 高 | 週次MTGで早期検知、課題即時解決、エスカレーション体制 |
| **技術的トラブル** | 中 | 高 | 24時間サポート体制、オンサイト対応、バックアップ環境 |
| **セキュリティ事故** | 低 | 極高 | ISO27001準拠、監査、インシデント対応計画 |
| **期待値ギャップ** | 高 | 中 | キックオフで成功基準明確化、週次での期待値調整 |
| **有償転換率低迷** | 中 | 高 | ROI可視化、経営層へのプレゼン、柔軟な価格設定 |

### 7.2 対策詳細

#### リスク1: 参加企業の離脱

**早期警告サイン**:
- 週次MTGの欠席
- Slack応答の遅延
- Agent利用頻度の低下

**対策**:
1. 週次MTGでのヘルスチェック（Green/Yellow/Red判定）
2. Yellow判定企業には追加サポート投入
3. Red判定企業には経営層エスカレーション

#### リスク2: 技術的トラブル

**想定トラブル**:
- Agent実行エラー
- GitHub連携不具合
- APIレート制限

**対策**:
1. 24時間サポート体制（Slack、電話）
2. エスカレーションフロー（L1サポート → L2エンジニア → CTOまで）
3. バックアップ環境の準備

#### リスク3: セキュリティ事故

**想定シナリオ**:
- APIキー漏洩
- 不正アクセス
- データ漏洩

**対策**:
1. ISO27001準拠のセキュリティ体制
2. 定期監査（四半期ごと）
3. インシデント対応計画（CSIRT体制）
4. 顧客への即時通知プロセス

## まとめ

本パイロット導入計画は、カクシンの既存顧客との強固な関係性を活かし、Miyabi AI自律開発プラットフォームの**プロダクトマーケットフィット（PMF）を検証**するための戦略的プログラムです。

### 成功の鍵

1. **厳格な顧客選定**: 成功確率の高い20社に絞り込み
2. **手厚いサポート**: CSM 5名体制、週次MTG、24時間サポート
3. **定量的効果測定**: KPI駆動、ROI可視化
4. **迅速なフィードバック対応**: 週次での改善サイクル
5. **営業資産化**: ケーススタディ、推薦状の戦略的取得

### 期待成果

- **有償転換**: 10社以上、初年度ARR 1億円
- **ケーススタディ**: 5業界 × 1社 = 5本
- **プロダクト改善**: 50件以上のフィードバック、ロードマップ反映
- **ブランド構築**: 大手企業ロゴ、推薦状による信頼性向上

**「最小の資本と、最小の人の命の時間で、最大の付加価値を創造する」**

この理念を、20社のパイロット顧客とともに実証します。
