# SaaS α版ローンチ計画書
## Miyabi AI自律開発プラットフォーム α版ローンチ戦略

**作成日**: 2024年11月30日
**バージョン**: 1.0.0
**対象**: SaaSプラットフォーム先行ユーザー（目標100社）
**プロジェクト期間**: 2025年Q3（7-9月想定）
**親Issue**: #2 カクシン進化

---

## エグゼクティブサマリー

本計画書は、Miyabi AI自律開発プラットフォームのSaaS α版ローンチに関する包括的な戦略を定義します。パイロットプログラム（Issue #12）で得られた知見を活かし、先行ユーザー100社を獲得することを目標とします。

### 目的

1. **市場投入（GTM）**: SaaS市場への本格参入、ブランド認知度向上
2. **先行ユーザー獲得**: 100社の有償契約獲得（ARR目標: 2億円）
3. **プロダクト検証**: α版フィードバックによるβ版（Q4）への改善
4. **コミュニティ構築**: Miyabiユーザーコミュニティの形成
5. **営業基盤確立**: セールスプロセス、カスタマーサクセス体制の確立

### 期待成果

| 成果指標 | 目標値 | 測定方法 |
|---------|--------|---------|
| **先行ユーザー獲得** | 100社 | 契約数 |
| **初年度ARR** | 2億円 | MRR × 12 |
| **Webサイト訪問** | 月間10,000 PV | Google Analytics |
| **リード獲得** | 500社 | CRM登録数 |
| **転換率（Lead→Customer）** | 20%以上 | CRM分析 |
| **NPS** | +40以上 | 顧客アンケート |

## 1. α版機能スコープ

### 1.1 MVPコア機能（Must Have）

#### A. 7つの自律型Agent

| Agent | 機能概要 | α版対応範囲 |
|-------|---------|-----------|
| **CoordinatorAgent** | タスク分解・並列実行制御 | DAG構築、Critical Path特定 ✅ |
| **IssueAgent** | Issue分析・ラベル管理 | 自動分類（65ラベル体系） ✅ |
| **CodeGenAgent** | AI駆動コード生成 | TypeScript生成、Claude Sonnet 4統合 ✅ |
| **ReviewAgent** | コード品質判定 | 静的解析、品質スコア80点基準 ✅ |
| **PRAgent** | PR自動作成 | Draft PR作成、Conventional Commits ✅ |
| **DeploymentAgent** | CI/CDデプロイ | GitHub Actions統合、Rollback機能 ✅ |
| **TestAgent** | テスト自動実行 | Vitest統合、カバレッジ80%目標 ✅ |

#### B. プラットフォーム基盤

| 機能 | 詳細 | α版ステータス |
|------|------|--------------|
| **マルチテナント対応** | Schema-per-Tenant（PostgreSQL） | ✅ 実装済み |
| **認証・認可** | OAuth2/OIDC、JWT | ✅ 実装済み |
| **API Gateway** | Kong、Rate Limiting | ✅ 実装済み |
| **Agent実行環境** | Kubernetes Namespace分離 | ✅ 実装済み |
| **監査ログ** | 全API呼び出し記録 | ✅ 実装済み |
| **暗号化** | TLS 1.3、AES-256（at rest） | ✅ 実装済み |

#### C. ユーザーインターフェース

| インターフェース | 機能 | α版ステータス |
|---------------|------|--------------|
| **Web Dashboard** | Agent実行状況、品質スコア可視化 | ✅ 実装済み |
| **CLI Tool** | `miyabi` コマンド（Issue作成、Agent実行） | ✅ 実装済み |
| **VS Code拡張** | Issue作成、コード生成統合 | 🔄 α版で限定提供 |
| **GitHub App** | Webhook統合、自動ラベリング | ✅ 実装済み |

---

### 1.2 α版除外機能（β版以降）

| 機能 | 除外理由 | 実装予定 |
|------|---------|---------|
| **SAML 2.0統合** | エンタープライズ専用、α版は中小企業中心 | β版（Q4） |
| **カスタムAgent SDK** | 高度な要望、パイロット後に優先度確定 | β版（Q4） |
| **マルチリージョン** | 初期は東京リージョンのみ | 2026年Q1 |
| **オンプレミス版** | 開発コスト高、需要は金融・公共のみ | 2026年Q2 |
| **多言語対応（英語以外）** | 日本市場優先 | 2026年Q3 |

---

### 1.3 α版の制限事項

| 制限項目 | 内容 | 理由 |
|---------|------|------|
| **同時実行Agent数** | 最大5並列 | インフラコスト削減 |
| **Issue処理数** | 50件/月（Starter）、200件/月（Growth） | リソース制限 |
| **ストレージ** | 10GB/テナント | コスト削減 |
| **SLA** | Best Effort（99%目標、保証なし） | α版のため |
| **サポート** | 営業時間内（平日9-18時）、Slack/Email | 24時間対応はβ版以降 |

### D. 紹介プログラム設計（案）

#### 概要

既存ユーザーが新規ユーザーを紹介することで、双方に特典を提供

#### インセンティブ

| 立場 | 特典 |
|------|------|
| **紹介者** | 紹介成功1社につき、次月利用料20%OFF |
| **被紹介者** | 初月無料（通常特典）+ 2ヶ月目も50%OFF |

#### 紹介フロー

1. 紹介者が管理画面から「紹介リンク」発行
2. 被紹介者がリンク経由で申し込み
3. 被紹介者が有償プラン契約
4. 紹介者に自動で特典適用

#### 上限

紹介者は月間5社まで（最大100%OFF相当）
